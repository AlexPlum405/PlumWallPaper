# PlumWallPaper 完整实现计划 (Phased Implementation) - v1.1

基于 PRD 审计与用户最新指令，更新以下阶段任务。

## Phase 1: 视觉补全与基础交互加固 (Data & UI Refinement)

**目标**：对齐 PRD 信息展示要求，确保首页基础交互无瑕。

- [ ] **信息补全**：首页 Section 卡片增加“视频时长”与“文件大小”标签。
- [ ] **Section 点击联动**：确保首页下方所有 Section 卡片点击后，顶部 Hero 区域同步切换图片与文本信息，并平滑置顶。
- [ ] **极致滚动体验**：确保所有超出窗口的 Section 均可横向滚动，但**必须完全隐藏滚动条**，优化触控板滑动摩擦感。
- [ ] **色彩调节补全**：面板增加“时长”信息，完善设置页缺失的“低电量/获取焦点”触发项。

## Phase 2: 核心闭环与多显示器逻辑 (Core Interactive Loops)

**目标**：实现 PRD 中强制要求的动态显示器分配流程。

- [ ] **动态显示器选项**：实现多显示器弹窗的动态生成逻辑。例如检测到 2 台显示器时，选项需具体化为：
  - 应用到所有显示器
  - 只应用到显示器 1 (Studio Display)
  - 只应用到显示器 2 (Dell U2723QE)
  - 分别为每个设置 / 取消
- [ ] **导入流程模拟**：实现点击“导入”后的文件选择模拟、进度条及 Toast 成功反馈。
- [ ] **色彩预设系统**：实现“保存预设”交互，弹出命名输入框并模拟存入列表。

## Phase 3: 全局高级交互 (Advanced UX & States)

**目标**：提升整体精致度，确保高级功能闭环。

- [ ] **右键菜单增强**：为壁纸库实现“编辑标签”浮动面板。
- [ ] **搜索闭环**：实现点击“搜索”后的全屏搜索层及实时结果筛选模拟。
- [ ] **适配模式联动**：在设置页切换适配模式（填充/居中等）时，实时改变背景预览效果。

## Phase 4: 最终交付与性能优化 (Final Polish)

**目标**：消除交互 Bug，达到演示级标准。

- [ ] 检查所有动画过渡（Easing），确保符合 macOS 原生质感。
- [ ] 检查并移除所有硬编码的 style，确保主题（浅色/深色）切换的一致性。
- [ ] 最终 Playwright 自动化测试验证所有跳转关系与状态机逻辑。
