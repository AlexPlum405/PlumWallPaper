> **文档版本**: v1.0

---

## 概述

当前 PlumWallPaper 使用 **AVPlayerLayer** 作为视频壁纸渲染引擎，适合标准视频播放场景。为支持以下高级功能，需要引入 **Metal 渲染引擎**作为第二渲染路径：

1. **实时生成内容**（粒子系统、程序化动画、Shader Art）
2. **极致自定义滤镜**（超越 Core Image 的多通道、时序效果）
3. **多视频混合**（画中画、分屏、实时抠像）
4. **交互式壁纸**（鼠标跟随、音频可视化）

---

## 架构设计：双引擎模式

### 核心理念

**保留 AVPlayerLayer 用于标准视频，新增 MetalWallpaperRenderer 用于高级功能**。两者通过统一的 `WallpaperRenderer` 协议共存。

```
WallpaperRenderer (协议)
├── BasicVideoRenderer (AVPlayerLayer) ← 现有实现
├── HEICRenderer (DesktopBridge) ← 现有实现
└── MetalWallpaperRenderer (Metal) ← 新增
```

### 为什么不全面迁移到 Metal？

| 场景 | AVPlayerLayer | Metal |
|------|--------------|-------|
| **标准视频播放** | ✅ 硬件加速解码、零拷贝、系统优化 | ⚠️ 需手动解码、内存拷贝、功耗高 |
| **实时生成内容** | ❌ 不支持 | ✅ GPU 计算、Shader 编程 |
| **多视频混合** | ❌ 单视频流 | ✅ 多纹理合成 |
| **交互式效果** | ❌ 静态播放 | ✅ 实时响应输入 |
| **自定义滤镜** | ⚠️ Core Image 单通道 | ✅ 多通道、时序、反馈循环 |

**结论**：标准视频用 AVPlayerLayer（性能最优），高级功能用 Metal（能力最强）。

### 阶段二: 实时生成内容（3-4 周）

#### 2.1 粒子系统

**目标**：支持 10,000+ 粒子的实时模拟（雪花、星空、烟雾等）。

**技术方案**：
- **GPU 粒子**：使用 Metal Compute Shader 在 GPU 上更新粒子位置/速度
- **实例化渲染**：单次 draw call 渲染所有粒子
- **纹理图集**：粒子贴图打包减少状态切换

**文件结构**：
```
Sources/Core/MetalContent/
├── ParticleSystem.swift          # 粒子系统管理器
├── Shaders/
│   ├── particle_update.metal     # 粒子更新 Compute Shader
│   └── particle_render.metal     # 粒子渲染 Vertex/Fragment Shader
└── Presets/
    ├── SnowParticles.swift       # 雪花预设
    ├── StarField.swift            # 星空预设
    └── FireParticles.swift        # 火焰预设
```

**配置参数**：
```swift
struct ParticleConfig {
```

#### 2.2 程序化动画

**目标**：数学驱动的动态图案（波纹、分形、噪声场）。

**技术方案**：
- **Fragment Shader 全屏绘制**：每个像素根据时间和位置计算颜色
- **Simplex/Perlin 噪声**：内置噪声函数库
- **时间 Uniform**：传递 `currentTime` 驱动动画

**示例 Shader**（波纹效果）：
```metal
fragment float4 ripple_fragment(VertexOut in [[stage_in]],
                                 constant float &time [[buffer(0)]]) {
    float2 uv = in.texCoord * 2.0 - 1.0;
    float dist = length(uv);
    float wave = sin(dist * 10.0 - time * 3.0) * 0.5 + 0.5;
    return float4(wave, wave * 0.8, wave * 0.5, 1.0);
}
```

### 阶段三: 极致自定义滤镜（2-3 周）

#### 3.1 多通道滤镜管线

**当前限制**：Core Image 的 `AVVideoComposition` 是单通道、静态参数。

**Metal 方案**：
- **多 Pass 渲染**：Bloom = 高斯模糊（横向）→ 高斯模糊（纵向）→ 叠加
- **Ping-Pong Buffer**：两个纹理交替作为输入/输出，支持反馈循环
- **时序效果**：访问前一帧纹理实现运动模糊、拖尾

**示例：Bloom 滤镜**
```swift
class BloomFilter {
    func apply(input: MTLTexture, output: MTLTexture) {
        // Pass 1: 提取亮部
        let bright = extractBright(input)
        
        // Pass 2-3: 高斯模糊
        let blurred = gaussianBlur(bright, iterations: 2)
        
        // Pass 4: 叠加到原图
        composite(input, blurred, output)
    }
}
```

#### 3.2 滤镜预设库扩展

**新增滤镜类型**：
- **色差 (Chromatic Aberration)**：RGB 通道偏移
- **CRT 扫描线**：复古显示器效果
- **Glitch 故障艺术**：随机像素错位
- **油画风格化**：Kuwahara 滤镜
- **实时景深**：基于深度图的模糊

**UI 集成**：
- 色彩调节页新增"高级滤镜"分类
- 每个滤镜可调参数（强度、半径、频率等）

### 阶段四: 多视频混合（3-4 周）

#### 4.1 画中画 (Picture-in-Picture)

**场景**：主壁纸 + 小窗口视频（如右下角播放动画）。

**技术方案**：
- **双 AVPlayer 解码**：主视频和 PiP 视频各自解码
- **Metal 合成**：将两个 `CVPixelBuffer` 转为 `MTLTexture`，在 Fragment Shader 中混合
- **变换矩阵**：PiP 窗口的位置、缩放、圆角

```metal
fragment float4 pip_composite(VertexOut in [[stage_in]],
                               texture2d<float> mainTex [[texture(0)]],
                               texture2d<float> pipTex [[texture(1)]],
                               constant PiPConfig &config [[buffer(0)]]) {
    float4 mainColor = mainTex.sample(sampler, in.texCoord);
    
    // 判断当前像素是否在 PiP 区域
    if (isInsidePiP(in.texCoord, config.rect)) {
        float2 pipUV = mapToPiP(in.texCoord, config.rect);
        float4 pipColor = pipTex.sample(sampler, pipUV);
        return mix(mainColor, pipColor, pipColor.a);  // Alpha 混合
    }
    
    return mainColor;
}
```

#### 4.2 分屏 (Split Screen)

**场景**：左右或上下分屏显示两个视频。

**实现**：
- 根据 `splitRatio` 参数划分屏幕
- 每个区域采样对应视频纹理

#### 4.3 实时抠像 (Chroma Key)

**场景**：绿幕/蓝幕视频去背景，叠加到另一个视频上。

**技术方案**：
- **色度键控 Shader**：判断像素颜色是否接近目标色（绿色/蓝色）
- **边缘羽化**：平滑抠像边界
- **溢色校正**：去除绿色反光

```metal
float chromaKeyMask(float3 color, float3 keyColor, float threshold) {
    float dist = distance(color, keyColor);
    return smoothstep(threshold - 0.1, threshold + 0.1, dist);
}
```

### 阶段五: 交互式壁纸（2-3 周）

#### 5.1 鼠标跟随

**效果**：粒子被鼠标吸引/排斥，或光标周围产生涟漪。

**技术方案**：
- **全局鼠标监听**：`NSEvent.addGlobalMonitorForEvents`
- **传递鼠标位置到 Shader**：作为 Uniform 变量
- **粒子受力计算**：在 Compute Shader 中根据鼠标位置施加力

```swift
NSEvent.addGlobalMonitorForEvents(matching: .mouseMoved) { event in
    let mousePos = event.locationInWindow
    self.updateMouseUniform(mousePos)
}
```

```metal
kernel void update_particles_with_mouse(device Particle *particles [[buffer(0)]],
                                         constant float2 &mousePos [[buffer(1)]],
                                         uint id [[thread_position_in_grid]]) {
    float2 toMouse = mousePos - particles[id].position;
    float dist = length(toMouse);
    float force = 100.0 / (dist * dist);  // 反平方力
    particles[id].velocity += normalize(toMouse) * force * deltaTime;
}
```

#### 5.2 音频可视化

**效果**：根据系统音频输出实时生成频谱、波形动画。

**技术方案**：
- **音频捕获**：使用 `AVAudioEngine` 或 `ScreenCaptureKit` 捕获系统音频
- **FFT 频谱分析**：Accelerate 框架的 `vDSP_fft`
- **传递频谱数据到 GPU**：作为 1D 纹理或 Buffer

```swift
let fft = vDSP.FFT(log2n: 10, radix: .radix2, ofType: Float.self)!
fft.transform(audioBuffer, output: &spectrum)

// 传递到 Metal
let spectrumBuffer = device.makeBuffer(bytes: spectrum, length: ...)
```

**Shader 可视化**：
- **频谱柱状图**：每个频段对应一个柱子高度
- **波形粒子**：粒子大小/颜色随频率变化
- **径向脉冲**：低频驱动整体缩放

## 性能优化策略

### 1. 自适应质量

根据 GPU 负载动态调整：
- **粒子数量**：GPU 占用 >80% 时减半
- **渲染分辨率**：降至 720p 再上采样
- **帧率**：从 60fps 降至 30fps

### 2. 后台暂停

当壁纸被遮挡或用户全屏工作时：
- 停止 Metal 渲染循环
- 释放 GPU 资源
- 保留状态以便快速恢复

### 3. 内存管理

- **纹理池**：复用 `MTLTexture` 对象
- **命令缓冲区复用**：避免频繁创建
- **异步加载**：Shader 编译在后台线程

## 附录：关键代码片段

### A. Metal 渲染循环骨架

```swift
func renderFrame() {
    guard let drawable = metalLayer.nextDrawable(),
          let commandBuffer = commandQueue.makeCommandBuffer(),
          let renderPassDescriptor = currentRenderPassDescriptor else { return }
    
    renderPassDescriptor.colorAttachments[0].texture = drawable.texture
    
    guard let encoder = commandBuffer.makeRenderCommandEncoder(descriptor: renderPassDescriptor) else { return }
    encoder.setRenderPipelineState(pipelineState!)
    
    // 绑定顶点/纹理/Uniform
    encoder.setVertexBuffer(vertexBuffer, offset: 0, index: 0)
    encoder.setFragmentTexture(sourceTexture, index: 0)
    
    // 绘制
    encoder.drawPrimitives(type: .triangleStrip, vertexStart: 0, vertexCount: 4)
    encoder.endEncoding()
    
    commandBuffer.present(drawable)
    commandBuffer.commit()
}
```

### B. CVDisplayLink 驱动

```swift
func startDisplayLink() {
    CVDisplayLinkCreateWithActiveCGDisplays(&displayLink)
    CVDisplayLinkSetOutputCallback(displayLink!, { (_, _, _, _, _, context) -> CVReturn in
        let renderer = Unmanaged<MetalWallpaperRenderer>.fromOpaque(context!).takeUnretainedValue()
        renderer.renderFrame()
        return kCVReturnSuccess
    }, Unmanaged.passUnretained(self).toOpaque())

}
```
