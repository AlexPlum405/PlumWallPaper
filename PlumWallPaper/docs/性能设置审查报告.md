### 1.2 实时性能仪表盘
**前端位置**: `plumwallpaper.html:1817-1849`  
**功能**: 显示 GPU 渲染压力、动态帧率、系统内存占用  
**实现状态**: ⚠️ 部分实现（假数据）  
**后端支持**: 
- ✅ `PerformanceMonitor.getCurrentMetrics()` 存在
- ❌ **GPU Load 返回固定 0.0**（`PerformanceMonitor.swift:19`）
- ❌ **FPS 返回固定 60**（`PerformanceMonitor.swift:23`）
- ✅ 内存占用真实（通过 `mach_task_basic_info` 获取）

**严重问题**:
1. **GPU Load 完全未实现** - 返回硬编码 0.0，无法反映真实 GPU 压力
2. **FPS 完全未实现** - 返回硬编码 60，无法反映实际渲染帧率
3. **前端柱状图动画是假的** - `height: (idx === 0 ? h : (25 + Math.random() * 75)) + '%'`，只有第一个柱子用真实数据，其他两个用随机数

**竞品对比**:
- **Wallpaper Engine (Windows)**: 提供真实 GPU 使用率、VRAM 占用、实时 FPS
- **Lively Wallpaper (Windows)**: 显示真实 CPU/GPU/内存占用
- **Plash (macOS)**: 无性能监控（轻量级工具）

**建议**:
1. **立即修复**: 实现真实的 GPU Load 和 FPS 获取（使用 Metal Performance Shaders 或 CADisplayLink）
2. **或者删除**: 如果无法实现，删除这个假仪表盘，避免误导用户

### 1.3 智能暂停策略 (Smart Pause)
**前端位置**: `plumwallpaper.html:1851-1862`  
**功能**: 9 个自动暂停条件开关  
**实现状态**: ⚠️ 部分实现

| 设置项 | 前端 | 后端 Settings | PauseStrategyManager | 实际生效 |
|--------|------|---------------|----------------------|----------|
| 电池供电时暂停 | ✅ | ✅ | ✅ `isOnBattery()` 真实实现 | ✅ |
| 处于全屏应用时暂停 | ✅ | ✅ | ⚠️ `hasFullscreenApp()` 只检查自己的窗口 | ❌ |
| 壁纸被完全遮挡时休眠 | ✅ | ✅ | ❌ `isDesktopOccluded()` 返回 false | ❌ |
| 低电量模式开启时暂停 | ✅ | ✅ | ✅ `ProcessInfo.isLowPowerModeEnabled` | ✅ |
| 屏幕共享或录制时暂停 | ✅ | ✅ | ❌ `isScreenSharing()` 返回 false | ❌ |
| 笔记本盖子关闭时暂停 | ✅ | ✅ | ❌ `isLidClosed()` 返回 false | ❌ |
| 运行高负载应用时暂停 | ✅ | ✅ | ⚠️ `cpuUsage() > 80` 实现了但阈值固定 | ⚠️ |
| 失去焦点时自动暂停 | ✅ | ✅ | ✅ `!NSApp.isActive` | ✅ |
| 系统锁定/进入睡眠前 | ✅ | ✅ | ✅ `NSWorkspace.willSleepNotification` | ✅ |

**严重问题**:
1. **5 个功能是假开关** - 用户打开后没有任何效果（遮挡检测、屏幕共享、盖子关闭）
2. **全屏检测逻辑错误** - `hasFullscreenApp()` 只检查 PlumWallPaper 自己的窗口，应该检查系统所有窗口
3. **高负载应用检测不准确** - 只检测 CPU 总使用率 > 80%，无法识别特定应用（如 Final Cut Pro）

**竞品对比**:
- **Wallpaper Engine**: 提供应用黑名单（检测到特定进程自动暂停）
- **Lively Wallpaper**: 支持全屏检测、电池模式、高性能应用检测
- **macOS 原生**: 无类似功能

**建议**:
1. **立即修复或删除假功能**:
   - 删除"壁纸被完全遮挡时休眠"（macOS 无 API 可靠检测）
   - 删除"屏幕共享或录制时暂停"（需要屏幕录制权限，实现复杂）
   - 删除"笔记本盖子关闭时暂停"（macOS 无直接 API）
2. **修复全屏检测**: 使用 `CGWindowListCopyWindowInfo` 检测所有全屏窗口
3. **增强高负载检测**: 添加应用黑名单功能（参考 Wallpaper Engine）

### 2.2 PauseStrategyManager
**文件**: `Sources/Core/PauseStrategyManager.swift`  
**轮询机制**: 每 5 秒检查一次所有暂停条件

**问题**:
1. **轮询间隔过长** - 5 秒意味着用户切换到全屏应用后，壁纸最多 5 秒后才暂停
2. **无事件驱动** - 应该监听 `NSWorkspace.didActivateApplicationNotification` 等系统通知
3. **暂停/恢复无防抖** - 如果条件频繁变化，会导致频繁调用 `pauseAll()`/`resumeAll()`

**建议**:
```swift
// 改为事件驱动 + 防抖
private var debounceTimer: Timer?
private func evaluatePauseConditions() {
    debounceTimer?.invalidate()
    debounceTimer = Timer.scheduledTimer(withTimeInterval: 0.5, repeats: false) { [weak self] _ in
        // 实际暂停逻辑
    }
}
```

private var displayLink: CADisplayLink?
private var frameCount = 0

func startMonitoring() {

    displayLink?.add(to: .main, forMode: .common)

@objc private func tick(link: CADisplayLink) {
    frameCount += 1
